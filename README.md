# 微博超话批量签到工具

> 参考项目：[https://github.com/imhuimie/WeiBo](https://github.com/imhuimie/WeiBo)
> 作者：Szy & Hjz



## 项目简介

本项目是一款基于 Web 端获取用户 Cookie 后，实现微博超话批量自动签到的工具。通过简洁的交互流程获取用户微博账号对应的 Cookie 信息，无需手动逐个操作超话签到，大幅提升多超话账号的签到效率，适用于有批量超话签到需求的用户。



## 功能特性

1. 支持 Web 页面可视化获取用户微博 Cookie，操作简单易上手；

2. 实现多账号、多超话的批量自动签到；

3. 签到结果实时反馈，清晰展示每个超话的签到状态；

4. 轻量级架构，部署成本低，支持宝塔面板快速部署。



## 环境要求

- 服务器系统：Linux（CentOS 7+/Ubuntu 18.04+ 推荐）；

- 宝塔面板：7.0 及以上版本；

- PHP 版本：7.4 或 8.0（推荐7.4）；

- Python 版本：3.7+（若涉及后端脚本运行）；

- 依赖扩展：curl、mbstring、openssl（PHP 扩展）。



## 部署方法（宝塔部署）

为简化部署流程，本项目文档采用宝塔面板辅助部署，具体步骤如下：



### 步骤1：安装宝塔面板

参考宝塔官方文档完成面板安装：[宝塔面板官方安装教程](https://www.bt.cn/bbs/thread-19376-1-1.html)  

安装完成后，登录宝塔面板并完成初始化环境配置（如安装 Nginx、PHP、Python 等基础组件）。



### 步骤2：创建网站

1. 登录宝塔面板，进入「网站」模块，，选择点击「Python项目」，点击「添加项目」；

2. 填写项目名称，选择python环境，设置项目路径（如 `/www/wwwroot/weibo-sign`）

3. 填写域名（若无域名可填写服务器 IP），打开外网映射



### 步骤3：上传项目文件

1. 将本项目的所有文件压缩为 ZIP 包；

2. 进入宝塔「文件」模块，找到步骤2创建的网站目录（如 `/www/wwwroot/weibo-sign`），点击「上传」，选择项目 ZIP 包并解压；

3. 确保解压后文件目录结构完整，核心文件（如 index.php、sign_script.py 等）位于网站根目录。



### 步骤4：配置运行权限

1. 在宝塔「文件」模块中，选中网站根目录，右键点击「权限」；

2. 将目录权限设置为 `755`，文件权限设置为 `644`，所有者设置为 `www`（宝塔默认运行用户），点击「保存」。



### 步骤5：配置 Python 依赖（若有）

1. 进入宝塔「终端」模块，切换到网站根目录：  

```bash

cd /www/wwwroot/weibo-sign

```

2. 安装项目所需 Python 依赖：  

```bash

pip install -r requirements.txt

```



### 步骤6：访问与使用

1. 在浏览器中输入部署的域名/服务器 IP，进入工具页面；

2. 按照页面指引，获取并填写微博账号 Cookie；

3. 选择需要签到的超话，点击「批量签到」按钮，等待签到结果反馈。

### 获取 Cookie

1. 登录微博网页版 (weibo.com)
2. 按 F12 打开开发者工具
3. 切换到 Network 标签
4. 刷新页面，找到任意请求
5. 复制 Request Headers 中的 Cookie 值

或使用Weibo Cookie Helper（https://github.com/Troray/GetCookies）



## 注意事项

1. Cookie 包含用户账号敏感信息，请勿向他人泄露，建议定期更换；

2. 请勿频繁调用签到接口，避免触发微博风控机制；

3. 若签到失败，可检查 Cookie 有效性、网络环境或超话是否存在访问限制；

4. 本工具仅用于个人学习与自用，请勿用于商业用途，遵守微博平台使用规范。



## 常见问题

### Q1：Cookie 获取后提示无效？

A1：请确认 Cookie 未过期、获取时的账号未退出登录，且 Cookie 字段完整（包含 SUB、SUHB 等核心字段）。



### Q2：批量签到时部分超话失败？

A2：可能是该超话需要关注后才能签到，或微博临时限制了该账号的操作，可手动验证后重试。



### Q3：宝塔部署后访问页面空白？

A3：检查 PHP 版本是否符合要求、网站目录权限是否正确，或查看 Nginx 错误日志定位问题。
